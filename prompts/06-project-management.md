# Project Management Prompts

## Project Initiation

1. **Project Charter**
   Create a project charter for [project name]. Include business case, objectives, scope, stakeholders, constraints, assumptions, and high-level timeline.

2. **Project Scope Statement**
   Write a detailed scope statement for [project]. Define inclusions, exclusions, deliverables, acceptance criteria, and constraints.

3. **Stakeholder Register**
   Create a stakeholder register for [project]. Identify stakeholders, their interests, influence level, and engagement strategy for each.

4. **Requirements Gathering**
   Design a requirements gathering approach for [project]. Include who to interview, what to ask, and how to document and validate requirements.

5. **Feasibility Assessment**
   Conduct a feasibility assessment for [project/initiative]. Evaluate technical, financial, operational, and schedule feasibility with risks.

6. **Project Kick-off Checklist**
   Create a kick-off checklist for [project type]. Cover pre-meeting prep, meeting agenda, and post-meeting actions to launch the project well.

7. **RACI Matrix**
   Develop a RACI matrix for [project]. Identify who is Responsible, Accountable, Consulted, and Informed for key decisions and deliverables.

8. **Project Brief**
   Write a project brief for [project] for [audience]. Summarize purpose, approach, timeline, and what you need from them.

9. **Resource Planning**
   Create a resource plan for [project]. Identify required skills, estimated effort, resource assignments, and capacity gaps.

10. **Communication Plan**
    Develop a communication plan for [project]. Define what information, to whom, how often, and through what channel.

## Planning & Scheduling

11. **Work Breakdown Structure**
    Create a WBS for [project]. Break down all work into manageable components with clear deliverables at each level.

12. **Project Timeline**
    Create a high-level project timeline for [project] spanning [duration]. Include major phases, milestones, and key dependencies.

13. **Milestone Definition**
    Define milestones for [project]. Include what needs to be true at each milestone, acceptance criteria, and target dates.

14. **Dependency Mapping**
    Map dependencies for [project tasks/workstreams]. Identify critical path, external dependencies, and risks from dependency chains.

15. **Sprint Planning**
    Plan a [duration] sprint for [team/project]. Prioritize backlog items, estimate effort, and set realistic sprint goals.

16. **Release Planning**
    Create a release plan for [product/project]. Define what's included, timeline, dependencies, and go-live criteria.

17. **Gantt Chart Narrative**
    Describe the project plan in narrative form: [paste schedule]. Explain phases, major activities, and key dates for stakeholder understanding.

18. **Buffer Planning**
    Determine appropriate buffer for [project timeline]. Consider risk factors, dependencies, and where buffers should be placed.

19. **Resource Leveling**
    Review this resource allocation and recommend leveling: [paste resource assignments]. Identify over-allocations and solutions.

20. **Critical Path Analysis**
    Identify the critical path for [project]. Highlight which activities must stay on schedule and recommend monitoring approach.

## Execution & Monitoring

21. **Status Report Template**
    Create a project status report template for [project type]. Cover progress, accomplishments, risks, issues, and upcoming work.

22. **Weekly Status Update**
    Write a weekly status update for [project]. Include progress this week, plans for next week, and any blockers or needs.

23. **Progress Tracking Framework**
    Design a progress tracking framework for [project]. Define metrics, measurement frequency, and reporting format.

24. **Burn-down Analysis**
    Analyze this burn-down data and provide insights: [paste data]. Assess pace, project if on track, and recommend adjustments.

25. **Task Prioritization**
    Help prioritize these tasks: [list tasks]. Consider urgency, importance, dependencies, and effort for each.

26. **Daily Standup Format**
    Create an effective daily standup format for [team size/type]. Keep it brief while surfacing important information and blockers.

27. **Sprint Review Preparation**
    Prepare for a sprint review for [sprint]. Summarize completed work, demo talking points, and gather feedback questions.

28. **Team Capacity Assessment**
    Assess team capacity for [upcoming period]. Factor in availability, commitments, and sustainable pace.

29. **Velocity Calculation**
    Calculate and analyze team velocity from: [paste sprint data]. Recommend capacity for future sprint planning.

30. **Deadline Management**
    This project deadline is at risk. Analyze the situation and recommend options: [describe current state and deadline].

## Risk & Issue Management

31. **Risk Register**
    Create a risk register for [project]. Identify 10-15 risks with probability, impact, mitigation strategies, and owners.

32. **Risk Assessment**
    Assess this risk: [describe risk]. Determine probability, impact, exposure, and recommend mitigation approach.

33. **Issue Log**
    Create an issue log template for [project]. Include fields for tracking, escalation criteria, and resolution workflow.

34. **Issue Escalation**
    Write an issue escalation for [issue]. Include impact, what's been tried, what's needed, and recommended path forward.

35. **Risk Mitigation Planning**
    Develop mitigation plans for these risks: [list risks]. Include specific actions, timelines, and owners for each.

36. **Lessons Learned on Risk**
    Document lessons learned about risks from [project]. What risks materialized, which mitigations worked, what would you do differently?

37. **Contingency Activation**
    Write a plan for activating contingency for [risk that materialized]. Include decision criteria, actions, and communication.

38. **Early Warning Indicators**
    Define early warning indicators for [project risks]. What signals should trigger escalation or contingency?

39. **Risk Communication**
    Communicate this risk to stakeholders: [describe risk]. Frame appropriately, explain impact, and describe what we're doing about it.

40. **Post-Incident Analysis**
    Analyze what happened with [incident/issue]. Determine root cause, impact, response effectiveness, and prevention for future.

## Stakeholder & Team Management

41. **Stakeholder Update**
    Write a stakeholder update for [project] tailored to [specific stakeholder/group]. Focus on what they care about.

42. **Executive Briefing**
    Create an executive briefing on [project] for [executive]. Cover status, key decisions needed, and support required.

43. **Team Performance Feedback**
    Provide constructive feedback to the project team on [aspect of performance]. Balance recognition with improvement areas.

44. **Conflict Resolution**
    Analyze this project conflict and recommend resolution: [describe conflict]. Consider perspectives and project impact.

45. **Team Motivation**
    The project team is [fatigued/demotivated/stressed]. Suggest approaches to improve team morale while maintaining progress.

46. **Vendor Management**
    Create a vendor management approach for [vendor] on [project]. Cover communication, performance monitoring, and issue resolution.

47. **Distributed Team Coordination**
    Design coordination practices for a distributed project team across [locations/time zones]. Address communication and collaboration challenges.

48. **Decision Log**
    Create a decision log template for [project]. Capture what was decided, by whom, rationale, and implications.

49. **Project Governance**
    Define governance structure for [project]. Include decision rights, escalation paths, and meeting cadence.

50. **Handoff Documentation**
    Create handoff documentation for transitioning [project/phase] to [receiving party]. Include status, open items, and context needed.
